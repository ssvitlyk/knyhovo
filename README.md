# Knyhovo

Knyhovo — сервіс для пошуку паперових книг, порівняння цін у найбільших українських книгарнях та сповіщень про зниження ціни.

## Основні можливості MVP

- Пошук паперових книг українською мовою
- Порівняння цін між книгарнями
- Сторінка книги з усіма доступними пропозиціями
- Базовий Wishlist (створення та керування списком бажаних книг)
- Email-алерти про зниження ціни
- Історія зміни цін

## Принцип розробки

Knyhovo будується як окремий продукт із чистою архітектурою. Старий Telegram-бот використовується як джерело доменних знань, але не як архітектурний фундамент.

Використовуємо з Telegram-бота:

- знання про провайдерів і книгарні;
- edge cases пошуку;
- логіку скрапінгу, яку можна переосмислити;
- ідеї canonical matching / дедуплікації;
- wishlist і price alert сценарії;
- реальні помилки, які вже були знайдені.

Не переносимо напряму:

- Telegram-specific логіку;
- тимчасові костилі;
- хаотичну структуру;
- код без тестів як production-ready рішення;
- старі архітектурні рішення без ревʼю.

## Wishlist як окрема продуктова фіча

Wishlist розглядається не як допоміжна функція, а як окрема продуктова складова Knyhovo.

### MVP

- створення власного wishlist;
- додавання та видалення книг;
- встановлення цільової ціни для книги;
- email-сповіщення про зниження ціни.

### Майбутній розвиток

- можливість ділитися wishlist з іншими користувачами;
- публічні та приватні wishlist;
- спільні wishlist для сім'ї або друзів;
- рекомендації на основі wishlist;
- сповіщення про вихід нових книг улюблених авторів або серій;
- імпорт існуючих списків книг.

Wishlist повинен проєктуватися як самостійний модуль із власним PRD, API та сценаріями розвитку, навіть якщо в MVP реалізована лише базова функціональність.

## AI-oriented direction

Knyhovo будується з чистим data foundation — canonical book entities, price history, wishlist — що є необхідною передумовою для будь-якого AI-enhanced продукту.

AI не є частиною MVP scope. Після стабілізації даних відкриваються можливості:

- семантичний пошук книг (query understanding замість keyword matching);
- рекомендації на основі wishlist, серії та жанру;
- "що читати далі" на основі завершених книг;
- price-aware рекомендації ("схожа книга дешевша зараз").

Деталі — у [`docs/prd/ai-discovery.md`](docs/prd/ai-discovery.md) (future PRD stub).

## Claude Code workflow

- `CLAUDE.md` у корені — карта проєкту, не енциклопедія.
- `.claude/rules/*.md` — правила, які мають бути активні завжди.
- `agent_docs/*.md` — детальна документація, яку Claude читає тільки за тригерами "Читай коли...".
- `docs/prd/*.md` — PRD для великих фіч перед реалізацією.
- Код для великої фічі не пишеться до підтвердження PRD.

## Git workflow

- Conventional Commits
- Feature branches only
- PR only
- Never commit directly to `main`
- Merge тільки після зеленого CI

## Мінімальний CI

- TypeScript strict
- ESLint
- Tests
- Smoke tests

## Перший етап

1. Створити документаційну структуру репозиторію.
2. Додати root `CLAUDE.md`.
3. Додати `.claude/rules/`.
4. Додати `agent_docs/` з порожніми шаблонами і тригерами читання.
5. Створити `docs/prd/web-mvp.md`.
6. Створити окремий PRD для Wishlist як самостійної фічі.
7. Тільки після підтвердження PRD починати код.
