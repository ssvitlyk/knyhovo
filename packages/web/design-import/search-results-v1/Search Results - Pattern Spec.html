Knyhovo Search Results v1.0

Офіційна специфікація патерну сторінки результатів пошуку. Складена виключно з затверджених компонентів DS v1.0. Жоден захищений актив не змінено.

1 · Поведінка футера

Футер завжди присутній у всіх трьох станах сторінки — результати, завантаження, порожньо. Макет і вміст ідентичні між станами і темами.

СтанФутер
РезультатиЗавжди відображається під пагінацією
ЗавантаженняЗавжди відображається під сіткою скелетонів
ПорожньоЗавжди відображається під секцією порожнього стану

2 · Стани SearchBar

СтанОписРеалізація
Idle Порожнє поле — іконка пошуку + placeholder «Назва книги, автора або ISBN…» Стандартний <SearchBar /> без value
Typing Користувач вводить запит. Під SearchBar з'являється кнопка × Очистити запит (--text-muted, hover → --accent). Очищення скидає query і committed. <SearchBar value={query} /> + .results__search-clear коли query.length > 0
Loading Скелетон SearchBar — точна копія капсули .kn-field з теплими блоками. Замінює справжній SearchBar під час завантаження. <SearchBarSkeleton /> — reuses .kn-field class
Results Введений запит зберігається у полі після пошуку. Кнопка очищення видима. value={committed} — controlled component
Empty Введений запит зберігається точно як надруковано. Не очищується автоматично. Кнопка очищення видима. Ідентично стану Results — value={committed}

3 · Ієрархія метаданих BookCard

Заморожено. Ніколи не змінюйте пріоритет або візуальний акцент цих елементів.

ПріоритетЕлементСтиль
1 — PrimaryПоточна цінаВеликий serif, --accent — найпомітніший елемент картки після назви
2 — SecondaryСтара ціна (якщо є)Закреслений, --text-muted, менший розмір
3 — TertiaryНазва магазинуЗавжди --text-muted. Ніколи не виділяти, не використовувати акцентний колір, не конкурувати з назвою книги або ціною.

Бейдж призначається за пріоритетом: «Найкраща ціна» (tone="green") → «-N%» (tone="solid") → «Новинка» (tone="accent"). Максимум один бейдж на картку.

4 · Пагінація — офіційний патерн

Зібрано виключно з компонента Button + токенів. Жодної нової візуальної мови. Вигляд ідентичний у світлій і темній темах.

Алгоритм усічення (getPageItems)

  • Завжди показувати першу і останню сторінки.
  • Завжди показувати поточну сторінку.
  • Показувати одну суміжну сторінку до і після поточної, якщо можливо.
  • Якщо між відображуваними сторінками лише одна прихована — показати її замість «…».
  • «…» лише коли прихованих сторінок більше однієї.
  • Якщо загальна кількість сторінок ≤ 7 — показати всі без усічення.

5 · Режим перегляду

List View виключено з MVP. Grid View — єдиний підтримуваний макет у Search Results v1.0. Перемикач вигляду не відображається. Заморожено: нова версія, що додає List View, повинна окремо специфікувати адаптивну поведінку BookCard у горизонтальному списку.

МакетСтатус MVP
Grid View (сітка)✓ Єдиний затверджений режим v1.0
List View (список)Відкладено. Не реалізовувати без окремої специфікації.

6 · Стани завантаження — офіційний патерн

Принцип: очікування виглядає як «відпочинок у затишній книгарні». Жодних холодних сірих блоків і стандартних спінерів.

ПатернСпецифікація
SearchBar loading Використовує справжній клас капсули .kn-field — розміри, радіус 16px і тінь точно відповідають затвердженому SearchBar. Всередині: кругла іконка 20px, лінія запиту 38% ширини, блок кнопки 92px (--accent-weak). Фон: --surface-accent.
BookCard skeleton Дзеркалить анатомію BookCard 1:1 — обкладинка 84×122 (--radius-sm), зона бейджа 88×18, заголовок 72%, автор 45%, рядок ціна 96×24 + магазин 88×12. min-height: 156px → нульові зсуви макету при завантаженні даних. --radius-md, --shadow-sm, hairline-бордер.
Повне завантаження SearchBar skeleton + тулбар («Шукаємо найкращі ціни…») + сітка скелетонів у тому ж grid з gap: --space-5. Кількість скелетонів = perPage.
Анімація Стагований fade-in: 280ms ease-out, 50ms каскад між картками. Одноразовий — без нескінченних циклів. Статичні теплі блоки при prefers-reduced-motion. Жодного shimmer, жодного scale-пульсу.
Кольори поверхонь Світла тема: --surface-accent → тепло-паперовий #F4E7D3. Темна тема: той самий токен → #2A2521 (тепло-чорнильний). Ніколи cold-grey.

7 · Порожній стан — заморожено

Ілюстративний порожній стан є навмисним і підкреслює ідентичність бренду. Книговик виступає провідником, що допомагає читачам продовжити подорож. Не спрощувати, не мінімізувати.

ЕлементСпецифікація
МаскотЛупа (mascot-magnifier.png) у світлій темі · Ліхтар (mascot-lantern.png) у темній. Висота 230px (desktop) / 180px (≤900px). Ніколи не замінювати на іконки або generic-ілюстрації.
Ієрархія повідомлень1. Заголовок: «Книгу не знайдено» (--fs-h3, Lora). 2. Підтримувальний текст: «Спробуйте іншу назву або ISBN.» 3. Популярні запити: Chip-компоненти, клік ініціює новий пошук.
ТемиОбидві теми мають ілюстрацію маскота — лише поза / аксесуар змінюється.
ЗапитВведений запит зберігається в SearchBar точно як надруковано. Кнопка «× Очистити» доступна.

8 · Адаптивність

БрейкпоїнтПравила
Desktop ≥ 768px Двоколонкова сітка BookCard. Горизонтальні BookCard (обкладинка ліворуч). Повна пагінація в один рядок. SearchBar max-width 720px. Тулбар в один рядок (підсумок ←→ сортування).
≤ 900px Навігація хедера ховається (як на головній). Тулбар складається у стовпчик. Маскот: 180px.
Mobile < 768px Одна колонка. BookCard і скелетони переходять у вертикальну композицію через page-level CSS — компонент BookCard не змінюється. SearchBar на всю ширину. Чипи сортування переносяться природно. Пагінація переноситься, всі кнопки мін. 44×44px для комфортного тачу.

9 · Нотатки для Claude Code

Структура компонента

const { SearchBar, BookCard, Button, Badge, Chip, ThemeToggle } = KnyhovoDS;

<SearchResultsPage>
  <SiteHeader />                      // «Каталог» — активний пункт

  <main class="results">
    eyebrow + h1 (запит у <em> — Lora italic, var(--accent))

    <div class="results__search">
      {loading ? <SearchBarSkeleton /> : <SearchBar value={committed} />}
      {!loading && query && <button class="results__search-clear">× Очистити</button>}
    </div>

    toolbar: aria-live summary + Chip×3 (selected = active sort)

    {loading} → SkeletonCard × perPage (grid, same gap)
    {empty}   → EmptyState (mascot + message + Chip suggestions)
    {results} → BookCard grid + Pagination
  </main>

  <SiteFooter />                      // завжди присутній
</SearchResultsPage>

Ключові правила імплементації

  • Тема: лише data-theme на <html>; маскот: magnifier ↔ lantern; лого: light ↔ dark.
  • Ціна: формат «245 ₴» (пробіл перед ₴).
  • Плюралізація: 1 книга / 2–4 книги / 5+ книг; aria-live="polite" на підсумку.
  • Бейдж: пріоритет — cheapest → знижка → новинка; максимум один.
  • Пагінація: алгоритм getPageItems(current, total) — завжди перша+остання+поточна+±1; заповнення однієї прихованої сторінки замість «…».
  • Сортування/пошук: скидає сторінку на 1.
  • Footer: завжди рендериться — у всіх трьох станах.
  • Доступність: пагінація — <nav> + aria-current="page"; сортування — role="group" + aria-label; скелетони — aria-hidden="true".
  • Layout shift: скелетони мають min-height: 156px — ідентичний реальним картками.

Заморожено разом із патерном:

  • Хедер і навігація (активний пункт: «Каталог»)

  • Композиція порожнього стану і використання маскота (ілюстративний, навмисний — не спрощувати)
  • Кольорова палітра, типографіка, відступи, тіні, бордери
  • Всі затверджені бренд-активи v1.0 (логотип, маскот)
